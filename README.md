# Urban-Insights-Dashboard

## Objetivo (ODS)

O projeto será desenvolvido em alinhamento com o ODS 11 – Cidades e Comunidades Sustentáveis, proposto pela Organização das Nações Unidas em https://brasil.un.org/pt-br/sdgs, e tem como objetivo tornar dados urbanos mais acessíveis, compreensíveis e úteis para a população e gestores públicos.

A proposta busca contribuir para:

- Melhor planejamento urbano
- Transparência de dados públicos
- Apoio à tomada de decisão baseada em dados

## Problema

Atualmente, dados urbanos importantes (como poluição, transporte e segurança) são:

- Dispersos em diferentes fontes
- Pouco intuitivos (planilhas, relatórios técnicos)
- Difíceis de interpretar para usuários comuns

Isso gera:

- Baixa utilização dos dados pela população
- Dificuldade na análise por gestores públicos
- Decisões menos eficientes

## Solução

A Solução proposta será de desenvolver um dashboard interativo que centraliza e apresenta dados urbanos de forma visual, simples e acessível.

Funcionalidades principais:

- Visualização de dados em gráficos interativos
- Filtros por região, período e tipo de indicador
- Comparação entre indicadores
- Interface amigável e responsiva

Indicadores abordados:
- Qualidade do ar (poluição)
- Transporte público (tempo médio, disponibilidade)
- Segurança (criminalidade)
- Saneamento (água e esgoto)

### Benefícios Esperados

- Melhor compreensão de problemas urbanos
- Apoio à tomada de decisão
- Maior transparência de dados públicos
- Incentivo ao uso de dados na sociedade

## Requisitos do Projeto

### Requisitos Funcionais

RF01 - O sistema deve exibir indicadores de qualidade do ar  
RF02 - O sistema deve apresentar gráficos de dados urbanos  
RF03 - O usuário deve poder filtrar dados por período  
RF04 - O sistema deve exibir dados de diferentes regiões  
RF05 - O sistema deve permitir atualização dos dados  

### Requisitos Não Funcionais

RNF01 - O sistema deve ser responsivo   
RNF02 - O tempo de carregamento deve ser rápida  
RNF03 - A interface deve ser intuitiva e de fácil uso  
RNF04 - O código deve ser versionado no GitHub

## Gerenciamento do Projeto

O projeto será gerenciado utilizando GitHub Projects, adotando uma abordagem ágil baseada em Kanban.

As dasfuncionalidades foram organiza em um backlog inicial, derivado dos requisitos funcionais. As atividades são distribuídas em sprints, sendo o TP2 correspondente à primeira sprint de desenvolvimento.

O fluxo de trabalho segue as etapas:
Backlog → TODO → In Progress → Done.
